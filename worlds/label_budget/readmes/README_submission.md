# Label-budget (active learning): submission set

One row per built task with its student score **band**. The product is the band, never the mean (see
`../../../CLAUDE.md`). The budget here is a **total label budget L** (buy training labels), not a per-row
feature budget, this is the first built commit-mode scheme (roadmap item 2). Sibling world:
per-row feature acquisition (`../../budgeted/readmes/README_submission.md`).

- **spread** = band width (max − min of the non-degenerate runs).
- **#band_supports** = tiers the band CAN hold = `1 + spread / LSD`, `LSD = z·√2·σ` (z=2), σ the test-set
  resampling std capped by the rarest test class. **The submit decision uses #band_supports only.**

**Decision rule:** `#band_supports ≥ 3` = SUBMIT-viable; `≤ 2` = REJECT. But a resolvable band is NOT
sufficient: it must also show a **skill gradient / strategy diversity** in the shipped code, not one
converged recipe (this world's central lesson, below).

Every number is produced by [`../../../scratch/analysis/band_resolution.py`](../../../scratch/analysis/band_resolution.py).
Each row links to its full record under [`tasks/`](tasks/).

## The label-budget lesson (carry into every scheme)

**label-budget-covtype** is the exception that taught the rule: its band IS resolvable (#band_supports
6.82), but it is eliminated for a reason the band alone can't see, every student converged on the same
acquisition recipe, so there is no strategy diversity to grade. A HIGH #band_supports can still be a bad
task if the students converge on ONE strategy. **Pair the band check with a strategy-diversity check**
(read what the 5 shipped solutions actually did).

Its salvage **label-budget-covtype-open** (hints stripped so the recipe is no longer handed over,
`pool_per_class` starving the rare classes) then produced a real, resolvable **skill gradient**
(#band_supports 7.82, driven by rare-class recall, legible in the submitted code), 4 tiers occupied, not
one outlier. It is a skill gradient within one recipe family, not yet strategy diversity, but it is the
first label-budget row that SUBMITs on merit. The remaining lever for genuine strategy diversity is the
label-cost second axis (design bible §21).

## Methodology

Same noise-floor method as the sibling world (`../../../sdk/methodology/noise_floor.md` §11–14;
`../../../sdk/hor_utils/{noise,resolution}.py`): decode `predictions_b64`, reconstruct graded labels via
`DataView`, stratified block-bootstrap σ → #band_supports / #observed / gap test. Records + this table
are GENERATED by `band_resolution.py --emit`; do not hand-edit rows, re-emit.
